## ⭐ Featured Projects

### 📦 Inventro

**Multi-Store Inventory and Point-of-Sale Platform**

Inventro is a full-stack retail inventory platform designed for companies managing products, staff, sales, purchases, and stock across multiple stores.

#### Key Features

- Multi-company and multi-store inventory management
- Product catalogue shared across company stores
- Barcode scanning and product lookup
- Sales and purchase workflows
- Stock ledger as the inventory source of truth
- Real-time inventory updates using WebSockets
- JWT authentication and secure API access
- Role-based access control
- Product image upload and management
- User profile, avatar, name, and biography management
- Store-level inventory search and availability
- Responsive Flutter interface for mobile devices
- PostgreSQL database and Supabase integration
- Backend and WebSocket deployment using Render

#### User Roles

- **Admin** — manages the complete platform
- **Owner** — monitors company stores and performance
- **Manager** — manages company stores and the product catalogue
- **Store Manager** — handles sales, purchases, and inventory for an assigned store

#### Architecture and Engineering

- Flutter Clean Architecture
- Riverpod state management and code generation
- Repository pattern
- RESTful API design
- Django REST Framework serializers and permissions
- Company- and store-scoped data access
- Django Channels for real-time communication
- PostgreSQL relational database design
- Supabase Storage for product and profile images
- Image processing with EXIF and GPS metadata removal

#### Technology Stack

`Flutter` `Dart` `Riverpod` `Dio` `GoRouter`  
`Django` `Django REST Framework` `Django Channels`  
`PostgreSQL` `Supabase` `JWT` `WebSockets`  
`Render` `Git` `Postman`
